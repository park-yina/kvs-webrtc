# 주요 요구사항
- 최대 3대가 동시에 들어와도 끊기지 않는 web-rtc가 가장 중요합니다.<br>
- 3개의 화면을 하나의 멀티뷰어로 볼 수 있게 하면서 1차적으로는 태블릿(lte환경)에서 지원하고 시간 내로 가능하다면, desktop도 원합니다.<br>
- 조이스틱을 활용하여 줌인 줌아웃 및 상하좌우 조절이 가능해야하며, 이에 대한 통신은 라즈베리파이에서 해석 가능하도록 꼭 정해진 통신 프로토콜 대로 전송해주세요<br>
- 마감 기한은 mvp모델은 3일, 최종 유저들이 들어오는 서비스까지 1주일입니다. 이에 염두해서 꼭 작업속도 맞추어주세요


# 디테일한 구현 사항 정리
##  조이스틱
- 조이스틱의 경우 가능하다면, 내가 클릭하는 곳이 어디든 그곳을 기점으로 조이스틱이 생기면 좋겠습니다.<br>
이는 상하좌우를 제어하는 조이스틱과 줌인/줌아웃을 제어하는 조이스틱 모두 동일합니다.

## 통신
- 통신의 경우 프로토콜을 맞추어주는 것이 1순위 입니다.<br> 또한 폴링을 사용하여 변동 사항이 없더라도 이후에 바로바로 체크가 될 수 있도록 원하는 전송속도에 맞추어서 전송을 해주어야합니다.
- mvp본은 wifi환경에서 작동해도 괜찮지만, 무조건 최종본은 lte환경에서 접속이 가능해야합니다. 이부분  꼭 여러번 부하테스트까지 완료해주세야합니다.
## ui
- 태블릿 화면의 가로방향을 기준으로 디자인을 해주세요.<br>
- 연령층이 다양한 만큼 전체화면 모드가 지원이 되면 좋겠고, 버튼이나 조이스틱 등이 캠을 보는 데에 방해가 되지 않도록 작업을 해주셔야합니다.
- 캠을 3개까지 연결 지원하므로, 2x2그리드에서 남는 한 개의 화면은 선택한 캠을 보여주는 화면으로 사용해주세요
